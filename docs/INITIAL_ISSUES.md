## 🎯 이슈 생성 순서

1. **초기 커밋 후**: 이슈 #1, #2, #3 (개선 이슈)
2. **기능 추가**: 이슈 #4, #5, #6 (기능 이슈)
3. **성능 개선**: 이슈 #7, #8 (성능 이슈)
4. **UX 개선**: 이슈 #9, #10 (UX 이슈)

## 📝 이슈 생성 방법

1. GitHub 저장소에서 "Issues" 탭 클릭
2. "New Issue" 클릭
3. 적절한 템플릿 선택 (Feature Request, Bug Report, Improvement)
4. 제목과 내용 작성
5. 라벨 추가
6. 이슈 생성

## 🔗 이슈와 PR 연결

각 이슈를 해결할 때:
1. 브랜치 생성: `git checkout -b feature/issue-4-s3-upload`
2. 작업 및 커밋: `git commit -m "feat: S3 이미지 업로드 연동\n\nRefs #4"`
3. PR 생성: PR 설명에 `Closes #4` 포함
4. 머지 시 이슈 자동 종료
