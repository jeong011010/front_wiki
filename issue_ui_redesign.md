# UI/UX 개선 및 반응형 디자인 구현

## 개요
프론트위키의 전체적인 UI/UX를 개선하고, 모든 디바이스에서 최적화된 반응형 디자인을 구현합니다. 사용자 경험을 향상시키고 현대적이고 일관된 디자인 시스템을 구축합니다.

## 목표
- ✅ 모든 페이지의 반응형 디자인 구현 (모바일, 태블릿, 데스크톱)
- ✅ 일관된 디자인 시스템 구축
- ✅ 사용자 경험(UX) 개선
- ✅ 접근성(Accessibility) 향상
- ✅ 현대적이고 세련된 UI 구현

## 작업 범위

### 1. 반응형 디자인 개선

#### 1.1 메인 페이지 (`app/page.tsx`)
- [ ] 모바일: 헤더 네비게이션 햄버거 메뉴로 변경
- [ ] 모바일: Featured Articles 카드 레이아웃 최적화 (1열)
- [ ] 태블릿: 2열 그리드 레이아웃
- [ ] 데스크톱: 3-4열 그리드 레이아웃
- [ ] 검색바 반응형 크기 조정
- [ ] 필터 바 모바일 최적화 (드롭다운 또는 모달)

#### 1.2 글 상세 페이지 (`app/articles/[slug]/page.tsx`)
- [ ] 모바일: 사이드바 제거 또는 하단으로 이동
- [ ] 모바일: 목차(TOC) 접기/펼치기 기능
- [ ] 태블릿: 2열 레이아웃 (본문 + 사이드바)
- [ ] 데스크톱: 최적 읽기 너비 유지 (max-width)
- [ ] 코드 블록 가로 스크롤 최적화
- [ ] 이미지 반응형 크기 조정

#### 1.3 글 작성/수정 페이지 (`app/articles/new/page.tsx`, `app/articles/[slug]/edit/page.tsx`)
- [ ] 모바일: 에디터 전체 화면 모드
- [ ] 모바일: 툴바 고정 또는 접기 기능
- [ ] 태블릿: 에디터 + 미리보기 분할 화면
- [ ] 데스크톱: 최적 편집 경험 제공

#### 1.4 지식 그래프 페이지 (`app/diagram/page.tsx`)
- [ ] 모바일: 그래프 줌/패닝 최적화
- [ ] 모바일: 컨트롤 버튼 접근성 개선
- [ ] 태블릿: 터치 제스처 지원
- [ ] 데스크톱: 키보드 단축키 지원

#### 1.5 로그인/회원가입 페이지 (`app/auth/login/page.tsx`, `app/auth/register/page.tsx`)
- [ ] 모바일: 폼 레이아웃 최적화
- [ ] 입력 필드 터치 영역 확대 (최소 44px)
- [ ] 에러 메시지 표시 개선

#### 1.6 관리자 페이지 (`app/admin/review/page.tsx`)
- [ ] 모바일: 테이블 카드 뷰로 전환
- [ ] 태블릿: 테이블 가로 스크롤 최적화
- [ ] 데스크톱: 전체 테이블 표시

### 2. 디자인 시스템 통일

#### 2.1 컴포넌트 스타일 통일
- [ ] 버튼 스타일 통일 (Primary, Secondary, Danger, Ghost)
- [ ] 카드 컴포넌트 통일 (`components/ArticleCard.tsx`, `components/AnimatedCard.tsx`)
- [ ] 입력 필드 스타일 통일
- [ ] 모달/다이얼로그 스타일 통일
- [ ] 배지(Badge) 스타일 통일

#### 2.2 색상 시스템

- [ ] 색상 대비율 WCAG AA 기준 충족
- [ ] 색상 팔레트 문서화

#### 2.3 타이포그래피
- [ ] 제목 계층 구조 명확화 (h1-h6)
- [ ] 본문 가독성 최적화 (line-height, letter-spacing)
- [ ] 모바일 폰트 크기 조정

#### 2.4 간격 시스템
- [ ] 일관된 spacing scale 적용 (4px, 8px, 16px, 24px, 32px, 48px, 64px)
- [ ] 컴포넌트 간 여백 통일

## 기술 스택

### 현재 사용 중
- **Tailwind CSS 4**: 스타일링
- **Framer Motion**: 애니메이션
- **React 19**: UI 라이브러리
- **Next.js 16**: 프레임워크

### 추가 고려사항
- **Radix UI**: 접근성 높은 컴포넌트 (선택사항)
- **Headless UI**: 접근성 컴포넌트 (선택사항)
- **React Hook Form**: 폼 관리 (이미 사용 중)

## 디자인 원칙

1. **모바일 퍼스트**: 모바일부터 디자인하고 확장
2. **일관성**: 모든 페이지에서 일관된 디자인 언어 사용
3. **접근성**: 모든 사용자가 접근 가능하도록
4. **성능**: 빠른 로딩과 부드러운 인터랙션
5. **가독성**: 명확한 타이포그래피와 적절한 여백

## 반응형 브레이크포인트

```css
/* Tailwind CSS 기본 브레이크포인트 */
sm: 640px   /* 모바일 가로 */
md: 768px   /* 태블릿 */
lg: 1024px  /* 데스크톱 */
xl: 1280px  /* 큰 데스크톱 */
2xl: 1536px /* 매우 큰 데스크톱 */
```
